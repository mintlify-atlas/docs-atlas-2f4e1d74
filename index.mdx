NestJS is a progressive Node.js framework for building efficient, scalable, and enterprise-grade server-side applications. It uses modern JavaScript, is built with TypeScript (while preserving compatibility with pure JavaScript), and combines elements of OOP, FP, and FRP.

Under the hood, NestJS makes use of [Express](https://expressjs.com/) by default, but also provides compatibility with [Fastify](https://www.fastify.io/), giving you access to the myriad of third-party plugins available for both platforms.

## Why NestJS?

NestJS provides an application architecture out of the box which allows for the effortless creation of highly testable, scalable, loosely coupled, and easily maintainable applications. The architecture is heavily inspired by Angular.

## Key features

<CardGroup cols={2}>
  <Card title="Modular Architecture" icon="puzzle-piece">
    Organize your application into self-contained modules that can be reused and shared across projects.
  </Card>
  <Card title="Dependency Injection" icon="syringe">
    Built-in IoC container makes it easy to manage dependencies and write testable code.
  </Card>
  <Card title="TypeScript First" icon="brackets-curly">
    Full TypeScript support with decorators and metadata reflection for expressive, maintainable code.
  </Card>
  <Card title="Platform Agnostic" icon="server">
    Supports both Express and Fastify adapters — swap HTTP platforms without changing application code.
  </Card>
  <Card title="Microservices" icon="network-wired">
    Built-in support for TCP, Redis, MQTT, NATS, RabbitMQ, Kafka, and gRPC transports.
  </Card>
  <Card title="WebSockets" icon="plug">
    First-class WebSocket support with Socket.io and ws adapters via the `@WebSocketGateway` decorator.
  </Card>
  <Card title="GraphQL" icon="diagram-project">
    Seamless GraphQL integration with code-first and schema-first approaches via `@nestjs/graphql`.
  </Card>
  <Card title="Testing Utilities" icon="vial">
    Built-in testing module that lets you mock providers and create isolated unit tests with ease.
  </Card>
</CardGroup>
