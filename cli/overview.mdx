The NestJS CLI (`@nestjs/cli`) is a command-line tool that helps you create and maintain NestJS applications. It scaffolds new projects, generates boilerplate files, builds your source, and starts the development server.

## Installation

Install the CLI globally so the `nest` command is available everywhere:

<CodeGroup>

```bash npm
npm install -g @nestjs/cli
```

```bash yarn
yarn global add @nestjs/cli
```

```bash pnpm
pnpm add -g @nestjs/cli
```

</CodeGroup>

Verify the installation:

```bash
nest --version
```

## Creating a new project

Run `nest new` followed by the project name to scaffold a fully configured NestJS application:

```bash
nest new my-app
```

The CLI will ask you to choose a package manager (npm, yarn, or pnpm) and then install all dependencies.

<Steps>
  <Step title="Run nest new">
    ```bash
    nest new my-app
    ```
  </Step>
  <Step title="Move into the project directory">
    ```bash
    cd my-app
    ```
  </Step>
  <Step title="Start the development server">
    ```bash
    npm run start:dev
    ```

    The application starts at `http://localhost:3000`.
  </Step>
</Steps>

## `nest-cli.json` configuration

The `nest-cli.json` file at the project root controls how the CLI compiles and serves your application.

```json nest-cli.json
{
  "$schema": "https://json.schemastore.org/nest-cli",
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "deleteOutDir": true
  }
}
```

### Key options

| Option | Type | Default | Description |
|---|---|---|---|
| `collection` | `string` | `"@nestjs/schematics"` | Schematics collection used by `nest generate`. |
| `sourceRoot` | `string` | `"src"` | Root directory of the application source. |
| `entryFile` | `string` | `"main"` | Entry file name (without extension). |
| `compilerOptions.deleteOutDir` | `boolean` | `false` | Delete the output directory before each build. |
| `compilerOptions.webpack` | `boolean` | `false` | Use webpack instead of `tsc`. |
| `compilerOptions.tsConfigPath` | `string` | `"tsconfig.build.json"` | Path to the TypeScript config file. |
| `projects` | `object` | вЂ” | Monorepo project definitions. |

## Available commands

| Command | Alias | Description |
|---|---|---|
| `nest new <name>` | `n` | Scaffold a new NestJS project. |
| `nest generate <schematic> <name>` | `g` | Generate a file using a schematic. |
| `nest build [app]` | | Compile the application with `tsc` or webpack. |
| `nest start [app]` | | Start the application (builds first). |
| `nest info` | `i` | Display system and NestJS package version info. |
| `nest add <library>` | | Install a library and run its setup schematic. |
