This guide walks you through installing the NestJS CLI, creating your first project, and running it locally.

## Prerequisites

- [Node.js](https://nodejs.org) version 20 or higher
- npm, yarn, or pnpm

## Install the NestJS CLI

The NestJS CLI scaffolds projects and generates application code for you. Install it globally:

<CodeGroup>
```bash npm
npm install -g @nestjs/cli
```
```bash yarn
yarn global add @nestjs/cli
```
```bash pnpm
pnpm add -g @nestjs/cli
```
</CodeGroup>

Verify the installation:

```bash
nest --version
```

## Create a new project

Run `nest new` and pick a name for your project:

```bash
nest new my-app
```

The CLI will prompt you to choose a package manager. Select npm, yarn, or pnpm.

Once complete, the CLI creates the following structure:

```
my-app/
├── src/
│   ├── app.controller.spec.ts
│   ├── app.controller.ts
│   ├── app.module.ts
│   ├── app.service.ts
│   └── main.ts
├── test/
│   ├── app.e2e-spec.ts
│   └── jest-e2e.json
├── nest-cli.json
├── package.json
├── tsconfig.build.json
└── tsconfig.json
```

| File | Purpose |
|------|---------|
| `src/main.ts` | Application entry point — bootstraps the Nest app |
| `src/app.module.ts` | Root module |
| `src/app.controller.ts` | A basic controller with a single route |
| `src/app.service.ts` | A basic service with a method |
| `nest-cli.json` | CLI configuration |

## Start the application

<Steps>
  <Step title="Navigate to the project directory">
    ```bash
    cd my-app
    ```
  </Step>
  <Step title="Start the development server">
    <CodeGroup>
    ```bash npm
    npm run start:dev
    ```
    ```bash yarn
    yarn start:dev
    ```
    ```bash pnpm
    pnpm run start:dev
    ```
    </CodeGroup>

    The `start:dev` command starts the app in watch mode, automatically restarting on file changes.
  </Step>
  <Step title="Test the endpoint">
    Open a new terminal and make a request to the default route:

    ```bash
    curl http://localhost:3000
    ```

    You should see the response:

    ```
    Hello World!
    ```
  </Step>
</Steps>

<Tip>
Use `npm run start:debug` to start with the Node.js debugger enabled, then attach your IDE's debugger to port 9229.
</Tip>

## Switch to Fastify (optional)

NestJS works with Express by default. To use Fastify instead, install the Fastify adapter:

<CodeGroup>
```bash npm
npm install @nestjs/platform-fastify
```
```bash yarn
yarn add @nestjs/platform-fastify
```
</CodeGroup>

Update `src/main.ts`:

```typescript src/main.ts
import { NestFactory } from '@nestjs/core';
import {
  FastifyAdapter,
  NestFastifyApplication,
} from '@nestjs/platform-fastify';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
  );
  await app.listen(3000);
}
bootstrap();
```

<Note>
By default, Fastify only listens on `127.0.0.1`. In Docker containers, use `app.listen(3000, '0.0.0.0')` to listen on all interfaces.
</Note>
